Design a complete cast of supporting characters for a novel.

## Input
The user will provide:
- A story premise or main goal (1-2 sentences)
- Genre (e.g., 奇幻冒險、都市修仙、科幻)
- Protagonist basics (name, personality, goal)
- Optionally: world setting details

If the user doesn't provide these, ask for them.

## Process

### Step 1: Analyze Story Needs
Based on the premise and genre, determine what character ROLES the story needs:
- **Mentor**: guides protagonist early, may die or betray later
- **Rival**: pushes protagonist to grow through competition
- **Antagonist**: main opposing force with understandable motivation
- **Ally/Partner**: fights alongside protagonist, provides emotional support
- **Love Interest**: emotional anchor, may create dilemmas
- **Comic Relief**: balances tension (optional, genre-dependent)
- **Mysterious Figure**: holds secrets, drives curiosity

### Step 2: Design Each Character
For each character, generate:

**Basic Info**:
- Name (fitting the world/culture)
- Age, gender, race (if fantasy)
- Role in story
- First appearance context (how they meet the protagonist)

**Depth**:
- Core motivation (what they WANT)
- Secret/hidden trait (what readers discover later)
- Fatal flaw (what causes their biggest mistake)
- Speaking style (specific patterns — formal, sarcastic, speaks in metaphors, stutters when nervous, etc.)

**Arc Design**:
- Starting state → Ending state (their transformation)
- Key turning point (the event that changes them)
- Relationship with protagonist evolution

### Step 3: Relationship Map
Create a relationship map showing:
- Who trusts whom (and why)
- Hidden tensions (who has conflicting goals)
- Potential betrayals or alliances
- Romantic triangles or rivalries

### Step 4: Dialogue Voice Test
For each major character, write a SHORT dialogue sample (2-3 lines) showing their unique speaking style in the same scenario: "Someone asks them for help."

This is the KEY validation — if all characters sound the same, the design failed.

## Output Format

Output a structured character bible in markdown with:
1. Character profiles (one section per character)
2. Relationship map (as a table or diagram)
3. Dialogue voice test (all characters responding to the same prompt)
4. Story integration notes (when each character should appear/exit)

## Quality Criteria
- ❌ FAIL if more than 2 characters have similar speaking styles
- ❌ FAIL if any character lacks clear motivation
- ❌ FAIL if relationship map has no conflict/tension
- ✅ PASS if each character could be identified by their dialogue alone
