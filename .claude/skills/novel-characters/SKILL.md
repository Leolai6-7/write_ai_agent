Design a complete cast of supporting characters for a novel based on $ARGUMENTS.

## Input Required
If not provided via arguments, ask the user for:
- Story premise / main goal (1-2 sentences)
- Genre (e.g., 奇幻冒險、都市修仙、科幻)
- Protagonist basics (name, personality, goal)
- World setting details (optional)

## Process

### Step 1: Analyze Story Needs
Determine what character ROLES the story needs:
- **Mentor**: guides protagonist, may die or betray later
- **Rival**: pushes protagonist through competition
- **Antagonist**: opposing force with understandable motivation
- **Ally/Partner**: fights alongside, emotional support
- **Love Interest**: emotional anchor, creates dilemmas
- **Mysterious Figure**: holds secrets, drives curiosity

### Step 2: Design Each Character (6-7 characters)
For each character:

**Basic Info**: Name, age, gender, race, role, first appearance context

**Depth**:
- Core motivation (what they WANT)
- Secret/hidden trait (what readers discover later)
- Fatal flaw (what causes their biggest mistake)
- Speaking style — MUST be very specific and unique:
  - BAD: "speaks politely" (too generic)
  - GOOD: "uses plant metaphors, pauses between sentences, switches to ancient elvish when nervous"

**Arc**: Starting state → Key turning point → Ending state

### Step 3: Relationship Map
Create a table showing:
- Who trusts/distrusts whom and WHY
- Hidden tensions and conflicting goals
- Potential betrayals or alliances

### Step 4: Dialogue Voice Test (CRITICAL)
ALL characters respond to the SAME scenario in 2-3 lines.
Scenario: "Someone asks them for help finding something important."

This is the KEY validation — if characters sound similar, the design FAILED.

## Output Format
1. Character profiles (one section per character)
2. Relationship map (as a table)
3. Dialogue voice test
4. Quality self-check

## Quality Criteria
- ❌ FAIL if 2+ characters have similar speaking styles
- ❌ FAIL if any character lacks clear motivation
- ❌ FAIL if relationship map has no conflict/tension
- ✅ PASS if each character identifiable by dialogue alone
