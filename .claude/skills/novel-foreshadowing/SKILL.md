Plan foreshadowing threads — when to plant seeds, drop hints, and deliver payoffs.

## Input

From $ARGUMENTS or conversation:
- **Story brief** (if available): determines scale and pacing
- **Story structure**: what arcs/chapters to plan foreshadowing for
- **Key plot points**: major events that need setup
- **Existing threads** (optional): unresolved threads from earlier chapters

## Philosophy

Foreshadowing is a promise to the reader. Every planted thread is a commitment to pay it off. The craft is in making the plant feel insignificant when it happens, and inevitable when it resolves.

### The 3-Beat Pattern
1. **Plant**: introduce the element naturally — readers barely notice
2. **Hint**: reference it in a different context — attentive readers start connecting
3. **Resolve**: pay it off — readers feel "of course, it was there all along"

Not all threads need all 3 beats. A minor detail can be planted and resolved without hints. A major mystery benefits from multiple hints across many chapters.

### Types
- **Major threads**: span large portions of the story, connect to central conflict
- **Minor threads**: resolve within a shorter span, enrich subplots
- **Red herrings**: deliberately mislead, revealed as misdirection — use sparingly, and make sure the misdirection itself serves the story

### Principles
- Major threads need enough hints that the resolution feels earned, not arbitrary
- A planted thread that goes too long without any reference risks being forgotten by the reader
- Plants should emerge from natural character behavior and plot events, not be inserted as "clues"
- Resolution should feel inevitable in hindsight but surprising in the moment
- Don't overload any single chapter with too many new threads — the reader needs breathing room

## Output Format

### Thread Plan

For each foreshadowing thread:
```
### [Thread Name]
- **Type**: major / minor / red_herring
- **Description**: one sentence
- **Related characters**
- **Plant**: how and where it's introduced naturally
- **Hints**: how it's referenced in different contexts
- **Resolution**: how it pays off, what the reader feels
- **Connection to main conflict**: how this thread serves the story
```

### Per-Chapter Directive Table

For the planned range, show what each chapter plants, hints, and resolves:
```
| Chapter | Plant | Hint | Resolve |
|---------|-------|------|---------|
```

## Quality Self-Check
- [ ] Every planted thread has a planned resolution
- [ ] Major threads have enough hints to feel earned
- [ ] Plants emerge from natural story events, not forced insertion
- [ ] Resolution feels inevitable in hindsight
- [ ] Thread density is manageable — reader can track what's active
